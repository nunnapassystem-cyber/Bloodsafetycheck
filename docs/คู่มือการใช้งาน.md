# คู่มือการใช้งาน ระบบตรวจสอบความปลอดภัยการให้เลือด
### SRK Safe Blood Transfusion System

---

## ส่วนที่ 1 — คู่มือสำหรับพยาบาล (User)

---

### 1. การเข้าสู่ระบบ

1. เปิด Browser บนมือถือหรือ Tablet → ไปที่ URL ของระบบ
2. กรอก **Email** และ **Password** ที่ได้รับจากผู้ดูแลระบบ
3. กด **เข้าสู่ระบบ**
4. ระบบจะแสดงหน้าหลักพร้อมชื่อพยาบาลและ Ward ของท่าน

> **หมายเหตุ:** ถ้าลืมรหัสผ่าน ติดต่อผู้ดูแลระบบ (Admin) เพื่อ Reset

---

### 2. ขั้นตอนการตรวจสอบก่อนให้เลือด

ระบบมี 3 ขั้นตอน ต้องทำตามลำดับทุกครั้ง

---

#### ขั้นตอนที่ 1 — Scan และระบุข้อมูลผู้ป่วย + ถุงเลือด

**2.1 Scan สติ๊กเกอร์ผู้ป่วย**

1. กด **"Scan ชื่อผู้ป่วย หน้าOrder สั่งให้เลือด"**
2. ให้กล้องจับ Barcode/QR Code บนแบบบันทึกการให้โลหิต
3. ระบบจะอ่าน HN อัตโนมัติ

**กรณี Scan ไม่ได้:**
- กด **"Scan ใหม่"** → ลองอีกครั้ง
- หรือกด **"พิมพ์ HN เอง"** → กรอก HN ด้วยมือ → กด **ยืนยัน**

**2.2 Scan ถุงเลือด**

1. หลังได้ HN แล้ว → กด **"Scan ถุงเลือด"**
2. ให้กล้องจับ Barcode บนถุงเลือด
3. ระบบบันทึกรหัสถุงเลือดอัตโนมัติ

**2.3 กรอกข้อมูลตรวจสอบ**

กรอกข้อมูลจาก **แบบบันทึกการให้โลหิต** และ **ป้ายถุงเลือด** ให้ครบ:

| ช่องข้อมูล | อ่านจาก |
|---|---|
| ชื่อ-สกุล ผู้ป่วย | แบบบันทึกการให้โลหิต |
| Blood Group ผู้ป่วย (ABO + Rh) | แบบบันทึกการให้โลหิต |
| ชนิดเลือดที่สั่ง (Order แพทย์) | แบบบันทึกการให้โลหิต |
| Blood Group ถุงเลือด (ABO + Rh) | ป้ายถุงเลือด |
| ชนิดเลือดในถุง | ป้ายถุงเลือด |
| ปริมาณ (ml) | ป้ายถุงเลือด |

4. กด **"ยืนยัน → ตรวจสอบ Blood Group + ชนิดเลือด"**

---

#### ผลการตรวจสอบ

**กรณี ✅ PASS (ข้อมูลตรงกัน)**

→ ไปขั้นตอนที่ 2 ได้เลย

**กรณี ❌ FAIL (ข้อมูลไม่ตรง)**

- ระบบแสดง **Alert สีแดง** พร้อมเสียงเตือน
- แสดงสาเหตุ เช่น "Blood Group ไม่ตรง: ผู้ป่วย A+ / ถุงเลือด B+"
- **ห้ามให้เลือดโดยเด็ดขาด**
- ระบบบันทึก FAIL log อัตโนมัติ
- กด **"ล้างข้อมูล เริ่มใหม่"** → ส่งถุงเลือดคืน Blood Bank

---

#### ขั้นตอนที่ 2 — ยืนยันด้วยพยาบาล 2 คน

1. ระบบแสดงสรุปข้อมูลก่อนยืนยัน → ตรวจสอบอีกครั้ง
2. **พยาบาลคนที่ 1** — แสดงชื่ออัตโนมัติจากการ Login
3. **พยาบาลคนที่ 2** — กรอกชื่อพยาบาลที่ยืนยันร่วม (ต้องเป็นคนละคนกับพยาบาลที่ 1)
4. กด **"ยืนยันเริ่มให้เลือด — บันทึกเวลา"**

---

#### ขั้นตอนที่ 3 — หน้าสรุปสำเร็จ

หลังยืนยัน ระบบจะ:
- แสดง **✅ บันทึกสำเร็จ** พร้อมเวลา
- แสดงสรุปข้อมูลการให้เลือดทั้งหมด
- บันทึก Audit Log อัตโนมัติ

**การบันทึกภาพ/แชร์ไป LINE:**
1. กด **"บันทึกภาพ / แชร์ไป LINE"**
2. มือถือจะเปิด Share sheet → เลือก LINE หรือ บันทึกรูป
3. ส่งเข้า Line กลุ่ม Ward ได้เลย

**เริ่มผู้ป่วยรายต่อไป:**
- กด **"เริ่มผู้ป่วยรายต่อไป"** → ระบบล้างข้อมูลและกลับสู่ขั้นตอนที่ 1

---

### 3. Session Timeout

- ถ้าไม่มีการใช้งาน **5 นาที** ระบบจะล้างข้อมูลผู้ป่วยอัตโนมัติ
- ต้องเริ่มกระบวนการใหม่ตั้งแต่ขั้นตอนที่ 1
- **ยังไม่ต้อง Login ใหม่** — เฉพาะข้อมูลผู้ป่วยที่ถูกล้าง

---

### 4. การออกจากระบบ

- กดปุ่ม **Logout** มุมบนขวา
- ควร Logout ทุกครั้งที่ใช้งานเสร็จ

---

---

## ส่วนที่ 2 — คู่มือสำหรับผู้ดูแลระบบ (Admin)

---

### 1. การเข้าสู่ระบบ Admin

1. ใช้ Email/Password Admin ที่ตั้งไว้ใน Supabase
2. หลัง Login จะเห็น **Admin Dashboard** แทนหน้าตรวจสอบเลือด

---

### 2. การดู Audit Log

**ดู Log ทุก Ward:**
1. เข้า Admin Dashboard → ส่วน **Audit Log**
2. เห็น Log ทุก Ward พร้อมคอลัมน์:
   - Ward, เวลา, Wristband ID, ถุงเลือด, ชนิด, Blood Group, ผลการ Match, พยาบาล 1, พยาบาล 2

**การ Filter:**
- **กรองตามวันที่** — เลือกวันที่ต้องการ
- **กรองตามผล** — เลือก PASS / FAIL / ทั้งหมด

---

### 3. การจัดการผู้ใช้ (User Management)

**เพิ่มพยาบาล/User ใหม่:**
1. เข้า [Supabase Dashboard](https://supabase.com/dashboard) → Project → **Authentication → Users**
2. กด **Add User → Create new user**
3. กรอก Email และ Password
4. หลัง Create แล้ว → คลิกที่ User → **Edit User Metadata** เพิ่ม:

```json
{
  "nurse_name": "ชื่อพยาบาล",
  "ward_id": "ward-A",
  "ward_name": "ศัลยกรรมชาย",
  "role": "nurse"
}
```

**รายการ ward_id ที่ใช้:**

| ward_id | ward_name |
|---|---|
| ward-A | ศัลยกรรมชาย |
| ward-B | ศัลยกรรมหญิง |
| ward-C | อายุรกรรมชาย |
| ward-D | กุมารเวชกรรม |
| ward-E | สูติกรรม |
| ward-F | อายุรกรรมหญิง |
| ward-G | วิกฤตศัลยกรรม |
| ward-H | วิกฤตอายุรกรรม |
| ward-I | วิกฤตอุบัติเหตุ, ศัลยกรรมหัวใจ |
| ward-J | ห้องผ่าตัดศัลยกรรม |
| ward-K | ห้องผ่าตัดสูติกรรม |
| ward-L | อุบัติเหตุชาย |
| ward-M | อาคาร1ชั้น4 |
| ward-N | อาคาร2ชั้น6 |

**เพิ่ม Admin ใหม่:**
```json
{
  "nurse_name": "ชื่อผู้ดูแล",
  "ward_id": "admin",
  "ward_name": "ผู้ดูแลระบบ",
  "role": "admin"
}
```

**Reset รหัสผ่าน:**
1. Supabase → Authentication → Users → คลิก User
2. กด **Send Password Reset Email** — ระบบส่ง link ไปที่ Email

**ลบ User:**
1. Supabase → Authentication → Users → คลิก User
2. กด **Delete User**

---

### 4. การดู Audit Log ใน Supabase (กรณีต้องการ Export)

1. Supabase Dashboard → **Table Editor → transfusion_logs**
2. กด **Export → CSV** เพื่อดาวน์โหลดข้อมูล

---

### 5. นโยบายข้อมูล

- Audit Log เก็บ **60 วัน** จากนั้นลบอัตโนมัติ (ทุกวันตี 4 น.)
- ระบบไม่เก็บชื่อผู้ป่วย — เก็บเฉพาะ Wristband ID (Barcode) เท่านั้น (PDPA)
- พยาบาลแต่ละ Ward เห็นเฉพาะ Log ของ Ward ตัวเอง
- Admin เห็น Log ทุก Ward
