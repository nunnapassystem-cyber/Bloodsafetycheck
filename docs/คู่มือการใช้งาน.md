# คู่มือการใช้งาน ระบบตรวจสอบความปลอดภัยการให้เลือด
### SRK Safe Blood Transfusion System — Version ล่าสุด (มิถุนายน 2569)

---

## ส่วนที่ 1 — คู่มือสำหรับพยาบาล (User)

---

### 1. การเข้าสู่ระบบ

1. เปิด Browser บนมือถือหรือ Tablet → ไปที่ URL ของระบบ
2. กรอก **Email** และ **Password** ที่ได้รับจากผู้ดูแลระบบ
3. กด **เข้าสู่ระบบ**
4. ระบบแสดงหน้าหลัก พร้อมชื่อพยาบาลและ Ward ของท่านที่มุมบนซ้าย

> **ลืมรหัสผ่าน:** กด "เปลี่ยนรหัสผ่าน" บนหน้าหลัก หรือติดต่อ Admin ให้ Reset

---

### 2. ขั้นตอนการตรวจสอบก่อนให้เลือด

ระบบแบ่งเป็น **3 ขั้นตอน** ต้องทำตามลำดับทุกครั้ง ไม่สามารถข้ามได้

```
ขั้นตอนที่ 1          ขั้นตอนที่ 2          ขั้นตอนที่ 3
ตรวจ Order +  ──→  ตรวจที่เตียงผู้ป่วย ──→  ยืนยัน 2 พยาบาล
ถุงเลือด
```

---

### ขั้นตอนที่ 1 — ตรวจ Order + ถุงเลือด

มี 4 หัวข้อย่อย (ก–ง) ต้องทำตามลำดับ

---

#### ก. ถ่ายรูปสติ๊กเกอร์ข้อมือ

1. กด **"ถ่ายรูปสติ๊กเกอร์ข้อมือ"**
2. กล้องเปิด → ถ่ายรูปสติ๊กเกอร์ที่ข้อมือผู้ป่วย
3. ปรับกรอบตัดให้ครอบข้อความ → กด **"ตกลง อ่านข้อความ"**
4. ระบบอ่าน HN และชื่อผู้ป่วยอัตโนมัติ

**ผลที่ได้:**
- แสดง **ชื่อ-สกุล** และ **HN** ในกล่องสีน้ำเงิน
- แก้ไขได้โดยตรงถ้า OCR อ่านผิด
- กด **"ถ่ายรูปใหม่"** ถ้าต้องการถ่ายใหม่

**กรณี OCR อ่านไม่ได้:**
- กด **"กรอก HN เอง"** → พิมพ์ HN จากสติ๊กเกอร์ → กด **"ยืนยัน HN"**

---

#### ข. กรอกข้อมูลจากแบบบันทึกการให้โลหิต

หลังได้ HN แล้ว กรอกข้อมูลจาก **แบบบันทึกการให้โลหิต** ให้ครบ:

| ช่อง | อ่านจาก |
|---|---|
| Blood Group ผู้ป่วย (ABO) | แบบบันทึกการให้โลหิต |
| Rh ผู้ป่วย (Positive/Negative) | แบบบันทึกการให้โลหิต |
| ชนิดเลือดที่สั่ง (Order แพทย์) | แบบบันทึกการให้โลหิต |

ระบบจะแสดง Blood Group สรุป เช่น **A+** หรือ **O-** หลังเลือกครบ

---

#### ค. ถ่ายรูปบัตรคล้องถุงเลือด

1. กด **"ถ่ายรูปบัตรคล้องถุงเลือด"**
2. ถ่ายรูปบัตรสีขาวที่คล้องอยู่กับถุงเลือด
3. ปรับกรอบตัดให้ครอบข้อความ → กด **"ตกลง อ่านข้อความ"**
4. ระบบอ่านข้อมูลถุงเลือดอัตโนมัติ

**ผลที่ได้ (แก้ไขได้ทุกช่อง):**
- ชื่อ/สกุล และ HN บนถุงเลือด
- ชนิดเลือด (PRC / LPRC / FFP / Platelet / WB)
- Blood Group และ Rh
- หมายเลขถุงเลือด
- ปริมาณ (mL)

> ถ้า OCR อ่านผิด → แตะที่ช่องนั้นแล้วแก้ไขได้เลย

---

#### ง. ตรวจสอบข้อมูลที่ตรงกัน

ระบบเปรียบเทียบข้อมูลจากสติ๊กเกอร์ข้อมือ vs บัตรถุงเลือดอัตโนมัติ:

| รายการ | หมายเหตุ |
|---|---|
| ✅ ชื่อ-สกุล ตรงกัน | ปลอดภัยดำเนินการต่อ |
| ⚠️ ชื่อ-สกุล ต่างกัน | ตรวจสอบอีกครั้ง (OCR อาจอ่านผิด) |
| ✅ HN ตรงกัน | ปลอดภัยดำเนินการต่อ |
| ❌ HN ไม่ตรงกัน | **ห้ามให้เลือด** ระบบบล็อกการดำเนินการ |

เมื่อข้อมูลตรงกัน → กด **"ยืนยัน → ตรวจสอบ Blood Group + ชนิดเลือด"**

---

#### ผลการตรวจสอบ Blood Group

**กรณี PASS (ข้อมูลตรงกัน)**

→ ระบบแสดง Checklist ก่อนออกไปเตียงผู้ป่วย:
- ✅ เตรียมป้ายมือให้เลือด
- ✅ เตรียมใบยินยอมให้เลือด

กดเครื่องหมายถูกทั้ง 2 ข้อ → กด **"รับทราบ — ไปขั้นตอน 2"**

**กรณี FAIL (ข้อมูลไม่ตรง)**

- ระบบแสดง **แถบแดง** พร้อมเสียงเตือนดัง
- แสดงสาเหตุชัดเจน เช่น:
  - `"Blood Group ไม่ตรง: ผู้ป่วย A+ / ถุงเลือด B+"`
  - `"ชนิดเลือดไม่ตรง: สั่ง PRC / ถุงเลือด LPRC"`
- **ห้ามให้เลือดโดยเด็ดขาด**
- ระบบบันทึก FAIL log อัตโนมัติ
- กด **"ล้างข้อมูล เริ่มใหม่"** → ส่งถุงเลือดคืน Blood Bank

---

### ขั้นตอนที่ 2 — ตรวจที่เตียงผู้ป่วย (ยืนยัน 2 พยาบาล)

1. ตรวจสอบสรุปข้อมูลทั้งหมดอีกครั้ง
2. **พยาบาลคนที่ 1** — แสดงชื่ออัตโนมัติจาก Login (ไม่ต้องกรอก)
3. **พยาบาลคนที่ 2** — กรอกชื่อพยาบาลที่มายืนยันร่วม
   - ต้องเป็น **คนละคน** กับพยาบาลที่ 1
4. กด **"ยืนยันเริ่มให้เลือด — บันทึกเวลา"**

---

### ขั้นตอนที่ 3 — บันทึกสำเร็จ

ระบบแสดง **✅ บันทึกสำเร็จ** พร้อมเวลาเริ่มให้เลือด และสรุปข้อมูลทั้งหมด

**ตัวเลือกหลังบันทึก:**

| ปุ่ม | ทำอะไร |
|---|---|
| **บันทึกภาพ / แชร์ไป LINE** | สร้างภาพสรุป → แชร์เข้า LINE กลุ่ม Ward |
| **ถุงเลือดถัดไป** | ข้ามไปสแกนถุงเลือดใบใหม่ (ผู้ป่วยคนเดิม) |
| **ผู้ป่วยรายต่อไป** | ล้างข้อมูลทั้งหมด เริ่มต้นใหม่ |
| **ออกจากระบบ** | Logout ออกจากระบบ |

---

### 3. Session Timeout

- ถ้าไม่มีการใช้งาน **5 นาที** ระบบล้างข้อมูลผู้ป่วยอัตโนมัติ (เพื่อความปลอดภัย PDPA)
- ต้องเริ่มกระบวนการใหม่จากขั้นตอนที่ 1
- **ไม่ต้อง Login ใหม่** — เฉพาะข้อมูลผู้ป่วยที่ถูกล้าง

---

### 4. เคล็ดลับการถ่ายรูปให้ OCR อ่านได้ดี

- ถ่ายในที่มีแสงเพียงพอ หลีกเลี่ยงเงา
- ถือมือถือนิ่ง ไม่เบลอ
- ปรับกรอบตัดให้ครอบเฉพาะตัวอักษร ตัดพื้นที่ว่างออก
- ถ้า OCR อ่านผิด → แก้ไขในช่องได้เลย ไม่ต้องถ่ายใหม่ทุกครั้ง

---

## ส่วนที่ 2 — คู่มือสำหรับผู้ดูแลระบบ (Admin)

---

### 1. การเข้าสู่ระบบ Admin

- ใช้ Email/Password ที่มี `"role": "admin"` ใน Supabase
- หลัง Login จะเห็นเมนูเพิ่มเติม: **Admin**, **Audit Log**, **เปลี่ยนรหัสผ่าน**

---

### 2. การดู Audit Log

ไปที่ **Audit Log** (บนหน้าหลัก)

**ฟีเจอร์:**
- กรองตาม **วันที่** — เลือกวันที่ต้องการ
- กรองตาม **ผล** — PASS / FAIL / ทั้งหมด
- **Export Excel** — ดาวน์โหลดข้อมูลเป็น .xlsx

**ข้อมูลในตาราง:**
Ward · เวลา · Wristband ID · ถุงเลือด · ชนิด · Blood Group · ผล (PASS/FAIL) · พยาบาล 1 · พยาบาล 2

**ยอด OCR Scans เดือนนี้** (แสดงด้านบนตาราง):
- ปกติ: `OCR Scans เดือนนี้: X / 1,000` (สีเทา)
- ใกล้ถึงลิมิต (≥ 900): แสดง **⚠️ ใกล้ถึงลิมิต** (สีเหลือง)
- เมื่อครบ 900 ระบบจะหยุด OCR อัตโนมัติ และแจ้งให้พยาบาลกรอก HN เอง

> Admin เห็น Log **ทุก Ward**, พยาบาลเห็นเฉพาะ Ward ของตัวเอง

---

### 3. การจัดการผู้ใช้ (User Management)

ไปที่ **Supabase Dashboard** → Project → **Authentication → Users**

---

#### เพิ่มพยาบาล/User ใหม่

1. กด **"Add User → Create new user"**
2. กรอก Email และ Password เริ่มต้น
3. คลิกที่ User ที่สร้าง → กด **"Edit User"** → แก้ไข **User Metadata** ดังนี้:

```json
{
  "nurse_name": "น.ต.หญิงนันท์นภัส ทองคำชุม",
  "ward_id": "ward-A",
  "ward_name": "ศัลยกรรมชาย",
  "role": "nurse"
}
```

**รายการ ward_id ที่ใช้ในระบบ:**

| ward_id | ward_name |
|---|---|
| ward-A | ศัลยกรรมชาย |
| ward-B | ศัลยกรรมหญิง |
| ward-C | อายุรกรรมชาย |
| ward-D | กุมารเวชกรรม |
| ward-E | สูติกรรม |
| ward-F | อายุรกรรมหญิง |
| ward-G | วิกฤตศัลยกรรม |
| ward-H | วิกฤตอายุรกรรม |
| ward-I | วิกฤตอุบัติเหตุ / ศัลยกรรมหัวใจ |
| ward-J | ห้องผ่าตัดศัลยกรรม |
| ward-K | ห้องผ่าตัดสูติกรรม |
| ward-L | อุบัติเหตุชาย |
| ward-M | อาคาร 1 ชั้น 4 |
| ward-N | อาคาร 2 ชั้น 6 |

---

#### เพิ่ม Admin ใหม่

```json
{
  "nurse_name": "ชื่อผู้ดูแลระบบ",
  "ward_id": "admin",
  "ward_name": "ผู้ดูแลระบบ",
  "role": "admin"
}
```

---

#### Reset รหัสผ่านพยาบาล

1. Supabase → Authentication → Users → คลิกที่ User
2. กด **"Send Password Reset Email"**
3. ระบบส่ง Link ไปที่ Email ของพยาบาล → คลิก Link → ตั้งรหัสใหม่

---

#### ลบ User

1. Supabase → Authentication → Users → คลิกที่ User
2. กด **"Delete User"**

---

### 4. การตรวจสอบ OCR Quota (Google Cloud Vision API)

**ดูยอดใช้งาน:**
- หน้า Audit Log → ดูแถบ **"OCR Scans เดือนนี้: X / 1,000"**
- หรือ Google Cloud Console → APIs & Services → Cloud Vision API → แถบ **Metrics**

**ระบบป้องกันอัตโนมัติ:**
1. ครบ 900 scan → ระบบหยุดส่ง OCR อัตโนมัติ พยาบาลต้องกรอกข้อมูลเอง
2. Budget Alert → ส่ง Email แจ้งเมื่อมีค่าใช้จ่าย
3. Quota reset ทุกวันที่ 1 ของเดือน

**ค่าใช้จ่าย:**
- 0–1,000 requests/เดือน = **ฟรี**
- เกิน 1,000 = ประมาณ $1.50 ต่อ 1,000 requests

---

### 5. นโยบายข้อมูล (PDPA)

- Audit Log เก็บ **60 วัน** → ลบอัตโนมัติทุกวันตี 4 น.
- ระบบเก็บเฉพาะ **Wristband ID** (Barcode) ไม่เก็บชื่อผู้ป่วย
- ข้อมูลผู้ป่วยอยู่ใน **RAM เท่านั้น** — ล้างทันทีเมื่อ session timeout หรือบันทึกสำเร็จ
- Google Cloud Vision API ไม่จัดเก็บภาพหลัง OCR เสร็จ
- พยาบาลเห็นเฉพาะ Log ของ Ward ตัวเอง (Row Level Security)
